1. 系统整体架构
本系统采用前端可视化页面 + 本地业务逻辑 + 远程大模型API三层轻量化架构，基于 Streamlit 快速开发，无后端独立服务，轻量化、易部署、易维护。
1.1 架构分层
1. 视图层（前端）：Streamlit 页面组件，包含输入框、按钮、标签页、分栏、文本展示、弹窗提示等交互组件
2. 业务逻辑层：会话状态管理、知识库解析、文本清洗、消息截断、参数校验、异常处理
3. 数据接口层：对接硅基流动大模型API，实现请求封装、响应解析、鉴权校验
1.2 项目目录架构
项目根目录
├── app.py          # 主程序（页面渲染、业务逻辑、API调用）
├── prompts.py      # 工具模块（知识库读取、系统提示词生成）
└── data/           # 知识库目录（存放所有校园MD文档）
    ├── 新生入学.md
    ├── 办事流程.md
    ├── 电话黄页.md
    ├── 应急防骗.md
    └── 交通出行.md
2. 核心模块设计
2.1 会话状态管理模块
全局初始化所有会话变量，保障页面刷新、按钮点击后数据不丢失，解决重复渲染、数据溢出问题。
核心存储内容：用户提问、AI回答缓存、多轮对话消息、问答历史记录、截断后的知识库文本、接口响应耗时。
2.2 知识库加载模块
遍历读取 data 目录下全部 MD 文件，合并为完整知识库文本，自动截断至6000字，避免上下文超长导致乱码、接口报错。支持根据用户身份动态生成差异化系统提示词。
2.3 交互页面模块
采用分区布局设计：顶部功能按钮区、身份选择区、快捷提问标签页、问答输入输出区、历史记录区、底部黄页信息区。所有快捷提问按钮支持一键填充提问并刷新页面，交互无延迟。
2.4 AI 问答调度模块
核心业务模块，包含参数校验、消息组装、上下文截断、API请求、响应解析、文本清洗、数据缓存全流程。自动控制对话上下文长度，避免多轮对话累计超限。
2.5 数据清洗模块
内置脏词过滤库、重复标签替换规则，自动清除AI返回的乱码、无效语句、重复知识库来源标记，统一回答展示格式，提升阅读体验。
2.6 日志与统计模块
记录每轮问答的时间、身份、问答内容，统计回答字数与接口响应耗时，可视化展示接口性能，支持历史记录回溯与清空。
3. 核心交互流程设计
3.1 系统启动初始化流程
1. 程序启动，初始化所有 Streamlit 会话状态变量
2. 自动读取本地 data 知识库文件，截断并缓存至会话
3. 渲染页面标题、功能按钮、身份选择、分类提问标签页
4. 加载底部校园电话黄页静态内容，完成页面初始化
3.2 常规问答交互流程（自定义提问）
1. 用户选择身份（新生/在校生/教师）
2. 用户输入自定义问题，系统同步更新会话提问内容
3. 校验提问长度，超长则弹出错误提示，终止后续流程
4. 检测本地知识库文件是否存在，缺失则提示异常
5. 组装系统提示词 + 历史对话 + 当前提问，截断超限上下文
6. 携带鉴权头部向大模型API发送请求，展示加载动画
7. 接收响应数据，处理超时、断网、鉴权失败等异常
8. 清洗回答文本，过滤脏数据、去重标签
9. 缓存回答内容、响应耗时，更新多轮对话上下文与历史记录
10. 页面渲染AI回答、统计字数与耗时，完成问答流程
3.3 快捷提问交互流程
1. 用户点击标签页内预设高频问题按钮
2. 系统自动将问题填充至输入框，清空旧回答缓存
3. 页面自动刷新，触发上述完整问答流程
3.4 新对话重置流程
1. 用户点击「开启新对话」按钮
2. 清空多轮对话消息、历史记录、提问内容、回答缓存
3. 页面刷新，恢复初始就绪状态，等待新提问
3.5 历史记录管理流程
- 每轮问答结束后，自动封装时间、身份、问答内容存入历史列表
- 页面倒序渲染所有历史记录，区分每轮问答
- 点击「清空历史」可一键重置历史记录数据
4. 接口设计
4.1 大模型请求接口
- 请求地址：https://api.siliconflow.cn/v1/chat/completions
- 请求方式：POST
- 请求头：Authorization 鉴权、Content-Type  JSON格式
- 请求参数：模型名称、系统提示词、历史对话、用户当前提问
- 超时时间：30秒
4.2 核心返回参数
解析 choices 数组内的AI回答内容，捕获 KeyError、IndexError 等格式异常，保障系统稳定。
5. 异常处理设计
系统全覆盖异常捕获，所有错误均提供人性化中文提示，无程序崩溃：
- 401错误：API密钥失效提示
- 超时错误：AI响应超时，建议精简问题重试
- 连接错误：网络连接失败提示
- 格式错误：AI返回数据异常提示
- 数据缺失：知识库文件缺失提示
- 超长提问：字符超限拦截提示
6. 页面布局设计
1. 顶部区：项目标题、新对话功能按钮
2. 用户配置区：身份选择下拉框、自定义提问输入框
3. 快捷提问区：三分标签页+双列快捷问题按钮
4. 问答展示区：AI回答内容、字数/耗时统计
5. 历史记录区：历史问答列表、清空历史按钮
6. 底部辅助区：校园官方电话黄页
