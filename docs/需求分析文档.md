
一、需求分析文档（SRS）
1. 项目概述
本项目为小航·郑州航院校园信息助手，基于 Streamlit 搭建轻量化 Web 问答系统，依托硅基流动大模型 API 与本地校园知识库，为郑州航院新生、在校生、教师三类用户，提供校园咨询问答服务。系统支持自定义提问、快捷分类提问、多轮对话、问答历史记录、AI回答优化清洗、接口性能统计等核心能力，解决校园信息查询零散、咨询效率低、新生入学疑问无统一解答渠道的问题。
2. 用户角色分析
系统面向三类核心用户，不同用户拥有差异化问答适配逻辑：
- 新生：核心需求为入学报到、缴费、宿舍、交通入校等新生指南类问题
- 在校生：核心需求为在读证明、校园卡补办、转专业、调课等校园办事流程
- 教师：核心需求为校园行政流程、应急处理、报销、校园运维等办公相关咨询
3. 功能需求
3.1 基础交互功能
- 身份选择：支持用户切换新生/在校生/教师身份，AI 根据身份差异化回复
- 自定义提问：支持文本输入校园相关问题，限制最大400字符，超长拦截提示
- 新对话重置：一键清空当前对话、历史记录、输入内容，初始化会话状态
3.2 快捷分类提问功能
通过标签页拆分三大场景，内置高频校园问题，一键自动填充提问并触发问答：
- 新生指南：报到流程、学费缴纳、宿舍环境、入校交通
- 办事流程：在读证明开具、校园卡补办、转专业、调课申请
- 应急防骗：冒充辅导员诈骗处理、设备报修、学生心理问题、差旅报销
3.3 AI 问答核心功能
- 知识库加载：自动读取本地 data 目录下所有 MD 知识库文件，截断超长文本防止上下文溢出
- 多轮对话：保存历史对话上下文，实现连续问答，自动截断超限对话记录
- AI 接口调用：对接硅基流动 DeepSeek-V4-Pro 模型，支持30秒超时保护
- 回答清洗过滤：自动过滤乱码、无效字符、重复知识库来源标记，优化回答展示效果
- 性能统计：实时计算AI响应耗时、统计回答字数并展示
3.4 历史记录管理功能
- 自动记录：保存每轮问答的时间、用户身份、提问内容、AI回答
- 倒序展示：最新问答优先展示
- 一键清空：支持手动清空全部问答历史
3.5 辅助信息展示功能
- 校园电话黄页：静态展示保卫处、校医院、招生办、后勤处等官方24小时联系电话
- 异常提示：覆盖API密钥失效、网络超时、接口报错、数据缺失、格式异常等全场景报错提示
4. 非功能需求
4.1 性能需求
- API 请求超时时间：30秒，超时自动提示重试
- 单条提问最大字符数：400字，避免超长提问导致接口报错
- 知识库单次加载最大长度：6000字，防止上下文溢出乱码
- 多轮对话最大消息限制：12条，控制上下文长度，保证响应速度
4.2 稳定性需求
- 完善异常捕获：支持超时、断网、接口401鉴权失败、数据格式错误等异常处理
- 数据容错：知识库文件缺失时主动弹窗提示，不崩溃退出
- 内容容错：自动清洗AI返回脏数据、重复标记、乱码字符
4.3 易用性需求
- 可视化操作：标签页、快捷按钮、分栏布局，操作零门槛
- 状态可视化：加载中提示、回答耗时/字数统计、清晰的报错文案
- 会话持久化：页面操作不丢失对话、历史记录
4.4 安全性需求
- 内容过滤：屏蔽违规、无效、垃圾文本输出
- 鉴权校验：识别失效API密钥并提示用户更换
5. 约束条件
- 系统依赖本地 MD 知识库文件，无知识库则无法正常问答
- 依赖外网大模型API接口，断网状态下无法使用AI问答功能
- 回答内容严格受限本地知识库，不支持知识库外的未知内容作答
